# Plan: Enhanced Globe Visualization and Obstacle Randomization

## Overview
This plan addresses three main requirements to enhance the UAV simulation:
1.  **Real Earth Map**: Replace the wireframe globe with a high-quality Earth texture.
2.  **Directional Lines**: Implement visual lines on the globe showing:
    *   The path/connection from the starting position to the current UAV position.
    *   The path/connection from the current UAV position to the destination.
3.  **Obstacle Randomizer**: Implement a system in the C++ engine to generate random obstacles in the UAV's flight path, forcing recalculation and avoidance.

## Detailed Tasks

### 1. Earth Texture Integration (Frontend)
*   **Asset Preparation**: Identify or download a suitable Earth texture (diffuse map) and potentially a bump/specular map for realism.
*   **Component Update (`Globe.tsx`)**:
    *   Modify the `Earth` component to use `useTexture` from `@react-three/drei`.
    *   Apply the texture to the `meshStandardMaterial` of the `Sphere`.
    *   Add a starfield background for better immersion.

### 2. Path Visualization (Frontend)
*   **State Expansion**: Update `UAVState` and the WebSocket communication to include `startPosition` and `destination`.
*   **Component Update (`Globe.tsx`)**:
    *   Implement a `PathLine` component using `THREE.Line` or `THREE.BufferGeometry`.
    *   **Line A (Start to Current)**: Draw a line from the initial takeoff coordinates to the current `uavState.lat/lon`.
    *   **Line B (Current to Destination)**: Draw a line from the current `uavState.lat/lon` to the target coordinates.
    *   Use `getCartesian` to ensure lines follow the sphere's curvature or are drawn as great-circle arcs (if complexity allows).

### 3. Obstacle Randomizer (Backend)
*   **Engine Update (`exec.cpp` & `Environment.h`)**:
    *   Implement an `ObstacleGenerator` class/logic.
    *   Create a mechanism to spawn `StaticObstacle` or `DynamicObstacle` at random coordinates within a reasonable bounding box of the UAV's trajectory.
    *   Update the main simulation loop to periodically check if new obstacles need to be added.
*   **Integration**:
    *   Ensure newly generated obstacles are broadcasted via the `MockSimulationServer` so they appear on the Web UI immediately.
    *   Verify that the `LocalAvoidance` logic in `exec.cpp` correctly detects and reacts to these new random obstacles.

## Implementation Steps
1.  **Phase 1: Visual Improvements (Frontend)**
    *   Update `Globe.tsx` with Earth textures.
    *   Implement the directional lines.
2.  **Phase 2: Dynamic Environment (Backend)**
    *   Modify C++ simulation loop to inject random obstacles.
    *   Test the interaction between random obstacles and the pathfinding/avoidance logic.
3.  **Phase 3: Verification**
    *   Verify the WebSocket correctly transmits the new obstacle data.
    *   Confirm the UI reflects the "dodge and recalculate" behavior.

## Success Criteria
*   The globe looks like a real Earth.
*   Lines clearly show the drone's journey (Start -> Current -> Target).
*   Obstacles appear randomly during simulation, and the drone's heading changes to avoid them.
