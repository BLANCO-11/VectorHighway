# Plan: Fix Earth Globe, UAV Links, and Velocity Slider

## Problem Description
The frontend application currently has the following issues:
1. The Earth globe appears as a dark abyss/black hole instead of showing landmasses.
2. The required visual links are missing:
    - An active line link between the UAV and its destination.
    - A passive line link representing the path the UAV is taking.
3. The Velocity control slider in the UI has a maximum limit that is too low (currently 1 km/s), whereas it should support up to 1000 km/s.

## Analysis

### 1. Earth Globe Issue
- I found two implementations of `Earth`:
    - `web/src/components/globe/Globe.tsx`: Uses a texture from `mrdoob/three.js`.
    - `web/src/app/page.tsx`: Uses a simple `meshStandardMaterial` with a dark blue color (`#1a3a5a`).
- In `web/src/app/page.tsx`, the `Earth` component being used in the main `SimulatorPage` does NOT use a texture, which explains why it looks like a dark sphere.

### 2. UAV Links Issue
- `web/src/components/globe/Globe.tsx` has logic to render lines (`start-to-drone` and `drone-to-target`).
- However, `web/src/app/page.tsx` (the main entry point) is NOT using the `Globe` component from `web/src/components/globe/Globe.tsx`. Instead, it defines its own local `Earth`, `UAVMarker`, and `ObstacleMarkers` components and does NOT implement any line rendering logic.

### 3. Velocity Slider Limit
- In `web/src/app/page.tsx`, the `ControlParams` interface and the `params` state define `speed` with a max of `1` in the input range.

## Proposed Plan

### Step 1: Unify Globe Implementation
- Replace the local `Earth` component in `web/src/app/page.tsx` with the more feature-rich `Globe` component from `web/src/components/globe/Globe.tsx`, or integrate the texture-based `Earth` into `page.tsx`.
- Ensure the `Earth` component correctly loads and displays the landmass texture.

### Step 2: Implement UAV Links in `page.tsx`
- Port the line rendering logic from `web/src/components/globe/Globe.tsx` to `web/src/app/page.tsx`.
- Define the two types of lines:
    - **Active line**: From current UAV position to the target destination.
    - **Passive line**: From the starting point to the current UAV position (representing the path taken).
- Use `THREE.BufferGeometry` and `THREE.LineBasicMaterial` to render these lines within the `Canvas` in `page.tsx`.

### Step 3: Increase Velocity Slider Limit
- Update the `ControlParams` and the `input` element in `web/src/app/page.tsx` to allow a maximum `speed` of `1000`.

### Step 4: Verification
- Verify that the Earth shows landmasses.
- Verify that the UAV's current position, target, and path are visually connected by lines as described.
- Verify that the velocity slider can be set to 1000.
